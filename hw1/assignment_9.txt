1. Представьте, что вы запускаете сервис cron. Напишите команду для проверки статуса работы сервиса. Какой результат она дает в вашей системе? Какие результаты могут быть?

Команда для проверки статуса: systemctl status cron

В моей системе результат обычно показывает "active (running)", что означает что сервис работает нормально. 

Другие варианты:
inactive (dead)         сервис остановлен
failed произошла        ошибка при запуске сервиса
masked                  сервис заблокирован и не может быть запущен
activating              сервис находится в процессе запуска
deactivating            сервис находится в процессе остановки


2. Что такое маскирование в systemctl? Как может использоваться этот функционал?

Маскирование - это более строгий способ отключения сервиса, чем обычный disable. Когда мы маскируем сервис командой systemctl mask service_name, systemd создает символическую ссылку юнит-файла на /dev/null. 

Это означает, что даже если какая-то другая служба попытается запустить замаскированный сервис по зависимости, это не получится. В отличие от disable, где сервис можно запустить вручную, при маскировании это невозможно.
Используется это в случаях, когда нужно полностью заблокировать сервис. Например:
Когда сервис конфликтует с другим и его нельзя запускать
Для предотвращения случайного запуска критических сервисов
Когда нужно временно отключить сервис, который может быть автоматически запущен другими службами


3. Как просмотреть зависимости конкретной службы с помощью systemctl?

Для просмотра зависимостей службы можно использовать несколько команд:
1. systemctl list-dependencies service_name показывает дерево зависимостей службы, какие службы она требует для работы
2. systemctl show -p Requires -p After service_name показывает более детальную информацию о зависимостях, конкретно поля requires (какие службы требуются) и After (в каком порядке должны запускаться)
3. systemctl list-dependencies --reverse service_name показывает обратные зависимости, то есть какие службы зависят от указанной
Например, для cron можно выполнить systemctl list-dependencies cron и увидеть все службы, которые нужны для его работы.
