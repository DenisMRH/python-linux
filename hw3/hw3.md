## Домашнее задание 3 по курсу "Python и основы Linux"

В этом задании предстоит спроектировать систему по сбору новостей из интернета.
Пусть у нас есть несколько поставщиков новостей, для тестирования рассмотрим следующие RSS-ленты:

* Интерфакс - https://www.interfax.ru/rss.asp
* Газета Коммерсант - https://www.kommersant.ru/RSS/main.xml
* Комменсант-Наука - https://www.kommersant.ru/RSS/science.xml
* Коммерсант-Политика - https://www.kommersant.ru/RSS/section-politics.xml
* МК-Политика - https://www.mk.ru/rss/politics/index.xml
* МК-Наука - https://www.mk.ru/rss/science/index.xml
* Коммерсант-СПБ - https://www.kommersant.ru/rss/regions/piter_all.xml
* АИФ-СПБ - https://spb.aif.ru/rss/all.php
* АИФ-РЗН - https://rzn.aif.ru/rss/all.php
* Инфо-РЗН - https://rss.rzn.info/rss/news/ryazan.xml


Из списка выше заметно, что RSS ленты даны в трех разрезах:
* Общие
* Тематические (Наука, политика, ...)
* Региональные (СПБ, РЗН, ...)

Требуется разработать систему, которая позволит выкачивать ленты новостей на основе правил, которые конфигурируются администратором системы.
Пример правила:
* Новости политики требуется выгружать из МК-Политика
* Новости по Санкт-Петербургу - из АИФ-СПБ
* Общие новости выгружать из Интерфакса и Коммерсанта

Для новости необходимы следющие поля:
* Таймстемп публицации
* Загаловок
* Описание
* Категория
* Таймстемп выгрузки в нашу БД

Прочие поля можно не рассматривать в рамках задания


Система будет состоять из двух компонент:
* Микросервис со следующим функционалом:
    1. Функционал администратора системы, где он может получать правила выгрузок и настраивать их (удалять, обновлят, добавлять новые)
    2. Функционал работы с новостями (возврат новостей за день, час, по определенному региону и другим фильтрам)
* Скрипта, который будет запускаться по расписанию и выгружать новостные ленты

В задании требуется создать только backend-часть работы данной системы. Создание пользовательского интерфейса не входит в рамки данного задания. 


Какие технологии требуется использовать:
* FastAPI для API
* Python-скрипт выгрузки новостных лент запускать в cron
* PostgreSQL в качестве базы данных (также можно использовать SQLite)
* Будет плюсом: Итоговое приложение должно запускаться как набор контейнеров в docker, используя docker-compose


Перед выполнением задания и написанием кода подумайте над следующими моментами:
1. Как хранить данные в нашей базе данных?
2. Как cron-джоба получает конфиги задачи?
3. Какие ручки должны быть у api администрирования?
4. Мы можем захотеть обновлять источники? Как мы сможем это делать?
